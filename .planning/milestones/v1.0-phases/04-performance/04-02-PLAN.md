```typescript
  q: z.string().max(500).optional(),
```

**In src/app/api/search/route.ts:**

```typescript

```

2. Calculate offset: `const offset = (page - 1) * limit;`

3. For the empty-params early return, update to include pagination metadata:

4. After building `whereClause`, add a COUNT query that uses the SAME whereClause and params:
```typescript
const countSql = `
  SELECT COUNT(*) as total
  FROM items i
  JOIN totes t ON i.tote_id = t.id
  ${whereClause}
`;
const { total } = db.prepare(countSql).get(...params) as { total: number };
```

5. Modify the data query to use parameterized LIMIT and OFFSET instead of hardcoded `LIMIT 100`:
```typescript
const sql = `
  SELECT i.*, t.name as tote_name, t.id as tote_id, t.location as tote_location
  FROM items i
  JOIN totes t ON i.tote_id = t.id
  ${whereClause}
  ORDER BY i.updated_at DESC
  LIMIT ? OFFSET ?
`;
const items = db.prepare(sql).all(...params, limit, offset) as Array<{...}>;
```

6. Update response to include pagination metadata:
```typescript
return NextResponse.json({
  data: {
    items,
    total,
    page,
    limit,
    total_pages: Math.ceil(total / limit),
  },
});
```

CRITICAL: The COUNT query and data query MUST use the exact same `whereClause` variable and `params` array (spread). Only the data query gets LIMIT/OFFSET appended. This prevents count drift (Pitfall 3 from research).

Also update the rawParams extraction to include page and limit from URL params:
