**Step 2: Create `src/lib/magic-bytes.ts` (VALID-02)**

Implement magic bytes detection for JPEG, PNG, and WebP. This is a standalone utility with no external dependencies.

```typescript
// Signatures to check:
// JPEG: first 3 bytes are FF D8 FF
// PNG:  first 8 bytes are 89 50 4E 47 0D 0A 1A 0A
// WebP: bytes 0-3 are 52 49 46 46 (RIFF) AND bytes 8-11 are 57 45 42 50 (WEBP)
//       (bytes 4-7 are file size, skip them)
```

Export two functions:
- `detectMimeType(buffer: Buffer): string | null` -- returns the detected MIME type or null
- `validateImageBuffer(buffer: Buffer): { valid: boolean; detectedType: string | null }` -- validates that buffer is one of the 3 allowed image types

**IMPORTANT (from research Pitfall 4):** WebP magic bytes are split -- RIFF at offset 0 and WEBP at offset 8. Must check BOTH locations. Only checking RIFF at offset 0 would match WAV and AVI files.

**Step 3: Create `src/lib/validation.ts` (VALID-01 foundation, VALID-03, VALID-04)**

Create the shared validation library with these exports:

1. **`parseJsonBody(request: Request)`** -- Safe JSON parse wrapper (VALID-04)
   - Calls `request.json()` in try-catch
   - Returns `{ data }` on success or `{ response: NextResponse }` with consistent 400 error on failure
   - Error response: `{ error: 'Invalid JSON in request body' }`

2. **`validateBody<T>(body: unknown, schema: T)`** -- Zod validation helper
   - Runs `schema.safeParse(body)`
   - Returns `{ data }` on success or `{ response: NextResponse }` with field-level 400 error on failure
   - Error response: `{ error: 'Validation failed', errors: [{ field, message }] }`
   - Field path from `issue.path.join('.')`

3. **`IdParam`** -- Zod schema for URL path parameters
   - `z.coerce.number().int().positive()` -- handles string-to-number conversion for route params

4. **`safePath(baseDir: string, userInput: string): string | null`** -- Path traversal prevention (VALID-03)
   - Resolves `path.resolve(baseDir, userInput)`
   - Returns null if resolved path doesn't start with `baseDir + path.sep` (or equal baseDir)
   - Used in import route for ZIP entry extraction

5. **Zod schemas for all input-accepting routes:**

   **Tote schemas:**
   - `CreateToteSchema`: name (string, min 1, max 255), location (string, min 1, max 255), size (string, max 100, optional), color (string, max 100, optional), owner (string, max 255, optional)
   - `UpdateToteSchema`: Same fields as Create but all optional (partial update). At least one field must be present -- add a `.refine()` that checks at least one key has a value.

   **Item schemas:**
   - `CreateItemSchema`: name (string, min 1, max 255), description (string, max 1000, optional), quantity (number, int, min 1, default 1)
   - `UpdateItemSchema`: name (string, min 1, max 255, optional), description (string, max 1000, optional), quantity (number, int, min 1, optional). At least one field must be present.

   **Metadata schemas:**
   - `CreateMetadataSchema`: key (string, min 1, max 255), value (string, min 1, max 1000)
   - `UpdateMetadataSchema`: key (string, min 1, max 255, optional), value (string, min 1, max 1000, optional). At least one field.

   **Action schemas:**
   - `MoveItemSchema`: tote_id (string, min 1, max 10)
   - `DuplicateItemSchema`: tote_id (string, min 1, max 10, optional -- defaults to same tote)
   - `BulkQrSchema`: tote_ids (array of strings, min 1 item, max 100 items)

   **Settings schema:**
   - `UpdateSettingsSchema`: settings (z.record(z.string(), z.string())) -- key-value object. No key whitelisting per HARD-01 (deferred to v2).

   **Search params schema:**
   - `SearchParamsSchema`: q (string, max 500, optional), location (string, max 255, optional), owner (string, max 255, optional), metadata_key (string, max 255, optional)

Use generous max lengths per research recommendation (255 for names, 1000 for descriptions) to prevent abuse without breaking legitimate use.

**Do NOT:**
- Create middleware (Next.js App Router doesn't support middleware for route handlers)
- Add key whitelisting for settings (HARD-01, deferred to v2)
- Use z.file() for photo uploads (FormData files aren't Zod-parseable; magic bytes handles this)
